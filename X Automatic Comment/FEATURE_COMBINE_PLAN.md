# X Automatic Comment - 双参考扩展融合功能梳理

更新时间：2026-03-28

## 1. 目标

在保留当前 `X Automatic Comment`（GasGx + Google + 星火）的基础上，融合两款参考扩展：

- 参考 A（GM Bot 11.1）：自动化流程与调度能力强
- 参考 B（ReplyX 1.1.6）：人设/互动模式/UI 交互与配置体验强

原则：
- 功能尽量 1:1 对齐参考能力
- 界面仅视觉主题改为“暗黑 + 绿色”，结构与操作路径尽量不改
- 中英文必须整块切换（中文全中文，英文全英文）

## 2. 三方能力矩阵

| 能力域 | 参考A GM Bot | 参考B ReplyX | 当前XAC状态 | 融合结论 |
|---|---|---|---|---|
| 自动化运行（启动/停止） | 有，支持 tab 级 start/stop | 有（偏自动回复） | 有 | 保留 XAC 实现，补齐参数与状态展示 |
| 自动滚动扫描 | 强（搜索页自动滚动+调度触发） | 有（自动回复链） | 有 | 采用 A 的调度思想 + 当前扫描实现 |
| 定时/概率调度 | 强（alarms + ScheduledStarts + probability） | 弱 | 部分（未完整暴露） | 引入 A 的可视化调度配置 |
| 自动动作编排（回复/点赞/关注/转推） | 强 | 以回复为主 | 以回复为主 | 新增动作开关与策略层（先回复，后点赞/关注/转推） |

## 3. 统一功能分层（融合后）

### 3.1 Runtime 自动化层（参考 A 为主）

- 搜索词路由：固定/自定义查询，进入 `latest/live`
- 自动滚动 + 扫描候选推文
- 动作编排链：
  1. 生成回复
  2. 写入编辑框
  3.（可选）发送
  4.（可选）点赞
  5.（可选）转推
  6.（可选）关注
- 定时槽调度：时间窗口 + 概率触发 + 自动恢复
- 运行指示器：当前状态、已完成数、上限、停止按钮

### 3.2 AI 与账号层（当前 XAC 为主）

- Google 登录（保留但不强制）
- GasGx/Supabase 会话同步
- 星火配置：本地保存 + 从 GasGx 同步
- 星火调用失败统一错误归一化（缺字段/超时/上下文失效）

### 3.3 交互与人设层（参考 B 为主）

- AI 配置文件（Profile）
  - 预设 + 新建 + 编辑 + 删除
  - 语气、目标、长度、语言、自定义指令、人设记忆、CTA
- 快速设置（会话覆盖）
- 互动模式切换（safe/spicy/viral）
- 自动发布开关、自动回复上限

### 3.4 UI/I18N 层（统一规范）

- 折叠态仅保留标题 + 展开按钮
- 展开态必须不溢出容器（details/select/textarea）
- 所有文案由 i18n 字典驱动，不允许硬编码混语
- 所有耗时点击必须给 loading（尤其 Google 登录、开始自动回复、生成）

## 4. 开发优先级（执行顺序）

### P0（先稳住可用性）

1. i18n 全量收口：下拉项、占位符、提示文案全部双语。
2. UI 溢出修复：星火配置展开、滚动区域、底部按钮固定布局。
3. loading 统一：按钮禁用+文案切换+状态条提示。
4. 非会员字数限制强制：生成后统一 `weighted <= 280`。

验收：
- 中文模式无英文残留；英文模式无中文残留。
- 任意点击耗时动作都有可见反馈。
- 回复框不会再出现超长负数提示。

### P1（能力对齐）

1. 动作编排开关：回复/点赞/转推/关注。
2. 调度中心：时间槽、概率、启动预览。
3. 结果统计：本轮执行数、动作分布、失败原因。

验收：
- 可设置“仅回复”或“回复+点赞+关注”等组合。
- 定时槽触发后可自动起跑。

### P2（增强体验）

1. Profile 模板库扩展（行业化模板）。
2. A/B 回复策略（保守/激进实验）。
3. 失败重试与风控节流（更细颗粒）。

## 5. 明确不做（当前阶段）

- 不接入第二套模型，统一使用星火。
- 不强制登录后才允许评论（登录开关保留）。
- 不改变核心布局路径，避免“看起来不是 1:1”。

## 6. 下一步（建议立即执行）

- 先做 P0 的第 1、2 项：
  1. i18n 全量收口（先修掉混语）
  2. 展开态溢出与滚动布局修正

完成后再进入 P0 第 3、4 项。
