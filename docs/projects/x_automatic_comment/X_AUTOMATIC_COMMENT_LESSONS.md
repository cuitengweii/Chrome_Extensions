# X Automatic Comment 经验教训（LESSONS）

## 2026-03-29：目录调整时必须先重新确认物理路径再继续写入
- 错误表现：线程中途工作区目录发生调整，若按旧路径继续落盘容易把事实写偏。
- 根因：默认沿用前一轮路径假设，未在继续执行前先做全目录复核。
- 更早发现方式：继续线程时第一步固定执行 `Get-ChildItem + git status --short` 对齐路径。
- 避免复发：所有归档与文件引用统一输出最新绝对路径，尤其是 `STATE` 的 `主路径` 字段。

## 2026-03-29：云备份回调要避免链式 `.catch` 直接挂在可能为空的返回值上
- 错误表现：极端并发点击时，`runPendingAction` 可能返回非 Promise，链式 `.catch` 存在调用风险。
- 根因：把控制流与异常处理混写在同一行。
- 更早发现方式：静态检查事件处理函数中的 `await fn().catch(...)` 模式。
- 避免复发：统一改为 `try/catch` 包裹异步 UI 事件主流程。

## 2026-03-28：状态更新未触发渲染导致“看起来没执行”
- 错误表现：点击停止或切换后，自动运行状态已变化，但按钮和面板文案未立即刷新。
- 根因：状态变量 `S.auto.active` 变更后缺少统一的 `render()` 调用。
- 更早发现方式：在功能测试中加入“状态变更即 UI 变更”的断言清单（启动/停止各一次）。
- 避免复发：凡是状态位影响按钮文案、样式或交互可用性，状态写入后必须触发渲染或统一状态机刷新。

## 2026-03-28：自动附加 GM/GN 造成搜索语义偏移
- 错误表现：用户输入 `(闲鱼 OR xianyu)` 后，系统自动拼接成 `(闲鱼 OR xianyu) (gm OR gn) ...`，导致结果异常或直接无结果。
- 根因：把历史默认词（GM/GN）硬编码进查询构建逻辑，覆盖了用户真实意图。
- 更早发现方式：在回归中加入“查询预览与最终 URL 完全一致”校验，并用真实中文关键词做样例。
- 避免复发：搜索构造器只拼“用户显式选择项”，禁止注入隐藏默认关键词。
